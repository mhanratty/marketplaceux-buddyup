Design Directions

After the prototype and results of user testing the SUMO team and the Firefox OS team discussed the following design directions

  • Live Help App - Build a live help app within the settings app where users can contact community members or operator support agents for help.
    • Pros: Trained community members and operator support agents can give more expert advice vs. friends/family
    • Cons: Difficulty in having enough community members/agents available in real-time to answer support requests, Scope of work to integrate with operator support systems, Not open enough for use cases outside of support
  • Feature of Loop - Incorporate screen sharing into Loop so that users can contact friends and contacts for help or other use cases.
    • Pros: Open to other uses cases than support, simplifies product/technology/maintenance by piggybacking on existing Firefox product/service
    • Cons: Discoverability of feature if it only appears on Loop, not possible or difficult to explain screen sharing could be used for help (and other) use cases
  • Screen sharing app - Building a screen sharing only app that would allow users to use screen sharing for a variety of use cases including help
    • Pros: Open to other uses cases than support, possible to explain screen sharing could be used for help (and other) use cases during app tutorial, Highly discoverable if it is a pre-loaded app on the phone
    • Cons: Another app on the phone, Maintenance of app over long term

Addressing some concerns...

  • Handling low number of support agents/community members
    • Present alternative of asking contact as a fall back if no community member/support agents are available
    • Instead of help at that moment, allow user to book a time with a community member like booking a time at the Apple genius bar. When the time happens community member/support agent will call the user.
    • Only offer community member/support agent live help option in locales with large volunteer base
    • Restrict live help times (9-5PM) and have community members/support agents sign-up for times
  • Improving discoverability
    • Live Help button in quick settings menu alongside options for airplane mode, wifi, etc.
    • Creating app pre-loaded on the device that appears on the app grid
    • Introducing features as part of 1st run experience. Replace tutorial with ability to connect to live help to receive tutorial.
